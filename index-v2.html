FAQ

Common Questions

What is Omnipotent?

Omnipotent is a Federated AI Sovereign Operating System with a kernel (Archon), persistent memory (Soul Journal), and constitutional rules (AI_RULES.md), all running on Docker. It coordinates 12 specialized AI agent personas through two closed-loop orchestration protocols with mandatory human checkpoints, a Signal Bus architecture, and tiered memory (Hot/Cold/Federated).

How do I install Omnipotent?

Run cargo install omnipotent-cli or download the pre-built binary from the releases page. Requires Docker 24+ and Deno 1.40+. Then run omnipotent init followed by omnipotent up.

Is Omnipotent free?

Source code is available on GitHub. Omnipotent is self-hosted -- no subscription fees, no cloud dependencies, no vendor lock-in.

What agents does it include?

12 specialized personas form The Legion: Archon (Kernel Architect), Oracle (Intel & Research), Shadow Route (Orchestration), Validitus (Security & QA), Codex (Truth & Doctrine), Eidolon (UI & Creative), Artificer (CI/CD & Deploy), Diplomat (Federation Sync), plus sub-roles Quartermaster, Scout, Mnemon, and Mercator.

Does Omnipotent require an internet connection?

No. Omnipotent runs fully offline once initialized. The Diplomat agent can optionally sync with federated instances via the opt-in Airlock Protocol, but it is never required.

How is this different from LangChain / CrewAI / AutoGen?

Omnipotent is a complete OS, not a framework. It includes infrastructure (Docker Swarm), a Rust CLI, persistent tiered memory, mandatory human checkpoints, and constitutional governance via AI_RULES.md. Frameworks give you building blocks. Omnipotent gives you a running sovereign.
